# ITVD Webinar: Microservices App (.NET, REST)

## 📌 Опис

Цей репозиторій містить приклад реалізації мікросервісної архітектури для вебінару:

**"Мікросервісна архітектура на .NET: REST, Docker та Kubernetes: від коду до кластера"**

У проєкті реалізовано два REST мікросервіси на платформі .NET, які взаємодіють між собою через HTTP.

Основна мета — продемонструвати базові принципи побудови мікросервісів та їх взаємодії у максимально зрозумілому та практичному вигляді.

---

## 🎯 Концепція рішення

Для спрощення архітектури та фокусування на ключових аспектах взаємодії між сервісами використано наступне обмеження:

> **Одне замовлення містить лише один товар**

Такий підхід дозволяє:

- уникнути складних вкладених структур даних
- не вводити додаткові сутності (наприклад, OrderItems)
- спростити модель даних
- зосередитися на міжсервісній взаємодії

Це типовий навчальний прийом, який дозволяє краще зрозуміти базові принципи без зайвої складності.

---

## 🏗️ Архітектура системи

Система складається з двох незалежних мікросервісів:

### 1️⃣ OrderService

Сервіс відповідає за створення та зберігання замовлень.

#### 📦 Модель даних

Використовується одна таблиця `Orders`, у якій кожен запис представляє замовлення.

#### Структура таблиці:

| Поле         | Тип       | Опис |
|--------------|----------|------|
| Id           | Guid     | Унікальний ідентифікатор замовлення |
| Status       | string   | Статус (Pending / Confirmed / Rejected) |
| ProductId    | int      | Ідентифікатор товару |
| ProductName  | string   | Назва товару |
| Price        | decimal  | Ціна на момент створення замовлення |
| CreatedAt    | DateTime | Дата створення |

#### 🧠 Архітектурні особливості

- Використовується **денормалізація даних** (ProductName, Price)
- Відсутні пов’язані таблиці (наприклад, OrderItems)
- Модель оптимізована для простоти та наочності
